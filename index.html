How it works

  1. Define the question. Keep it tight and decision-relevant.
  2. Upload sources. PDFs, notes, and datasets (later).
  3. Add your stance + constraints. What counts as success? What tradeoffs matter?
  4. Generate a brief. Claims must be supported by citations, or marked as uncertain.
  5. Reliability score. Based on citation coverage, evidence strength, and contradiction handling.
  6. 3-min explainer. Auto-generates a narrated script + slide plan (video rendering later).

Note: GitHub Pages hosts the UI. The “brain” (RAG + scoring) will be a small backend service.
